## 2. 핵심 설계 결정 (왜 이렇게 돼 있나)

- **타이머**: `경과 = 누적 + (now − 구간시작)` 모델(오차 없음). 운동=카운트업+**탭으로만 완료**, 휴식=카운트다운 후 **자동 진행 안 함**(초과시간 +표시, 3초 전 경고음).
- **유산소 분기**: `bodyPart === '유산소'` 하나로 전체 분기 — 세트표가 시간/거리 컬럼으로, 완료 시 스톱워치 경과 자동 기록. SetRecord에 `durationSec/distanceKm/avgHr/splitsSec/splitsHr`.
- **5×5 엔진**(data/fivexfive.ts): A/B 순서·다음 중량을 **저장하지 않고 완료 세션에서 매번 파생** → 운동을 놓쳐도 자동 보정, 기록 수정 시 추천 재계산. deload는 빈 바 미만 금지.
- **health.ts**: iOS+react-native-health 있을 때만 동작, 웹/Expo Go는 자동 비활성. 데모/검증용 모의: `globalThis.__fitHealthMock = {avgHr:142, distanceKm:1.5}`. 심박 존은 최대심박 190(30세) 고정 — 개인화(나이 입력)는 미구현 후보.
- **테마**: 화면 StyleSheet가 모듈 로드 시점 색을 캡처 → **index.ts에서 저장된 테마 적용 후 `require('./App')`** 하는 부트스트랩. 웹은 선택 즉시 reload로 적용, 네이티브는 재시작 안내. mutable `colors` + `applyTheme()`.
- **브랜드 기구**: 이름에 브랜드 프리픽스(`해머스트렝스 아이소레터럴 로우`) → 루틴/기록/통계 어디서든 자동 표시. Exercise.brand 필드 + 필터.
- **RPE 증량**(strength.ts `recommendNextWeight`): 지난 세션의 워킹 세트(워밍업 제외)가 전부 완료+목표 달성이고 RPE≤8(또는 미기록)이면 +2.5kg(lb는 +5). 완료 화면에서 운동별 RPE 칩(6~10)이 마지막 워킹 세트에 저장됨.
- **⚠️ WorkoutScreen 훅 함정**: `if (!meta) return` 조기 return이 있어서 **새 훅은 반드시 파생값 섹션이 아니라 상단 훅 섹션에** 추가할 것 (한 번 훅 순서 버그로 화면 전체가 깨진 적 있음).

## 3. 검증 워크플로 (이 대화에서 확립된 패턴)

1. `npx tsc --noEmit` → exit 0 필수
2. 웹 미리보기(`npm run web`, 포트 8081)에서: localStorage에 테스트 데이터 시드 → `location.reload()` → DOM 텍스트/입력값으로 동작 확인 → **테스트 데이터 반드시 정리**
3. 콘솔 에러 0 확인 후 커밋 → 즉시 push (커밋 메시지는 한국어 feat/docs/chore)
4. 워치 기능은 `__fitHealthMock`으로 모의 검증 (실기기 검증은 맥에서 처음으로 가능해짐!)
